## Análise de Arquitetura: React + PHP vs. Vanilla JS

### Opção A: React SPA (Vite) + TailwindCSS + PHP API (Recomendada)
- Admin como SPA em `admin-src/`, compilado no deploy via GitHub Actions
- Backend PHP refatorado como API JSON em `area-cliente/api/`
- UX premium sem recarregamentos, componentes modulares
- **Ponto de atenção:** Hostinger não roda Node — build acontece no GitHub Actions

### Opção B: Vanilla JS Moderno + TailwindCSS (Sem Build)
- Mantém PHP+HTML, mas limpa com TailwindCSS CDN e Fetch API
- Deploy direto, sem etapa de build
- Mais difícil gerenciar estados complexos (5 abas do cliente)

**Decisão proposta:** Opção A (React + Tailwind + PHP API)
