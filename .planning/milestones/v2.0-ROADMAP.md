# Roadmap — Vilela Engenharia (Milestone v2.0 Archive)

## Milestone v2.0 — Detalhes das Fases Concluídas

**Objetivo Geral:** Eliminar dívidas técnicas críticas de segurança, consolidar a arquitetura modular do admin (concluindo a Fase 09), unificar o design nas três camadas, tornar o admin interativo com Alpine.js, adicionar features de valor.

---

### Fase 1 — Segurança Crítica & Correção de Bugs 🔒
* **CSRF corrigido** nas 11 actions POST (SEC-06)
* **7 exclusões destrutivas** migradas GET → POST + CSRF (SEC-07)
* **Senha admin** hasheada com bcrypt; migração do valor em texto plano (SEC-08)
* `cliente_impersonate.php` refatorado + logout completo (SEC-09)
* Credenciais rotacionadas (SEC-10, operacional)
* Excluir cliente / excluir entregável / impersonate funcionando (ADM-16/17/18)

### Fase 2 — Consolidação da Arquitetura Admin 🧹
* `includes/processamento.php` eliminado; handlers em `actions/admin/` (ADM-19)
* Redirect via `header()` em `admin.php` (ADM-20)
* `cliente_detalhes.php` dividido em parciais por aba (ADM-21)
* Upload com validação de MIME real (ADM-22)

### Fase 3 — Sistema de Design Unificado 🎨
* Arquivo único de tokens (`tokens.css`) referenciado por landing, portal e admin (DES-01)
* Accent unificado (DES-02)
* Componentes padronizados conforme patterns (DES-03)
* Dark-mode coerente no portal (DES-04)

### Fase 4 — Admin Interativo (Alpine.js) ⚡
* Alpine.js via CDN na tela de 5 abas — estado de abas, ações sem reload (ADM-23)
* API JSON expandida (leitura/escrita) consumida pelo Alpine (ADM-24)
* KPIs do dashboard com atualização reativa (ADM-25)

### Fase 5 — Features de Valor do Admin ✨
* Exportação financeira Excel/CSV (FEAT-01)
* Tela de log de auditoria com filtros (FEAT-02)
* Busca global de clientes na sidebar (FEAT-03)
* Rastreamento de prazos da prefeitura (FEAT-04)
* Notas internas por cliente (FEAT-05)
* Alertas de pagamentos atrasados / prazos vencendo no dashboard (FEAT-06)
