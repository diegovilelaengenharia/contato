<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Secrets → runner CI/CD | Secrets são injetados no ambiente do runner — não aparecem nos logs |
| runner CI/CD → servidor Hostinger (FTP) | Arquivos gerados no runner (incluindo .env) são transferidos via FTPS |
| Internet → arquivos PHP/debug em produção | Arquivos de debug subidos para produção expõem lógica interna |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | Credenciais do banco expostas nos logs do GitHub Actions | mitigate | GitHub Secrets automaticamente mascarados nos logs (comportamento padrão do Actions — valores de secrets aparecem como "***") |
| T-03-02 | Information Disclosure | debug_*.php acessível em produção expõe lógica interna | mitigate | exclude list no FTP Action impede upload; validado via curl 404 após deploy (ASVS V14.3) |
| T-03-03 | Information Disclosure | session_test_*.php e probe.php expõem sessões PHP em produção | mitigate | Mesma exclude list do T-03-02 |
| T-03-04 | Tampering | .env gerado com valores incorretos se Secrets não estiverem configurados | mitigate | Checkpoint humano exige verificação dos 5 Secrets antes do push + teste de curl após deploy |
| T-03-05 | Denial of Service | Deploy com .env vazio quebra conexão PDO em todos os requests | mitigate | db.php (Plano 01) retorna HTTP 503 com mensagem genérica quando parse_ini_file() falha, evitando erro de stack PDO exposto |
</threat_model>
