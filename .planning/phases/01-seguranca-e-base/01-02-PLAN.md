Observações importantes antes de criar:
- Verificar se já existe um area-cliente/.htaccess (se existir, ler o conteúdo antes de sobrescrever)
- O arquivo pertence APENAS a area-cliente/ — não criar na raiz
- NÃO adicionar DirectoryIndex, RewriteBase, redirect HTTPS — já cobertos pelo .htaccess da raiz
- Usar sintaxe "Order Allow,Deny / Deny from all" (compatível com Apache 2.2 e 2.4 — padrão Hostinger)

Conteúdo exato do arquivo:

```apache
# Bloquear acesso HTTP direto a arquivos de configuração e credenciais

# Bloquear db.php e .env (credenciais e configuração sensível)
<FilesMatch "^(db\.php|\.env)$">
    Order Allow,Deny
    Deny from all
</FilesMatch>

# Bloquear diretório config/ inteiro (configurações internas)
<IfModule mod_rewrite.c>
    RewriteEngine On
    RewriteRule ^config/ - [F,L]
</IfModule>
```

Nada mais. O arquivo deve ter exatamente estas regras e nenhuma outra.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet → Apache (area-cliente/) | Requests HTTP externos podem tentar acessar diretamente arquivos de configuração PHP |
| Apache → PHP | Apache decide se processa o arquivo ou bloqueia antes de enviar ao PHP |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | area-cliente/db.php acessível via HTTP | mitigate | FilesMatch "^(db\.php\|\.env)$" com Deny from all bloqueia request antes de chegar ao PHP (ASVS V4.3) |
| T-02-02 | Information Disclosure | area-cliente/.env acessível via HTTP | mitigate | Mesmo FilesMatch do T-02-01 — .env e db.php bloqueados na mesma diretiva |
| T-02-03 | Elevation of Privilege | area-cliente/config/ acessível via HTTP (configurações internas) | mitigate | RewriteRule ^config/ - [F,L] retorna 403 para qualquer request ao diretório |
| T-02-04 | Information Disclosure | .htaccess raiz modificado acidentalmente e quebra HTTPS redirect | accept | Plano não toca o .htaccess da raiz — acceptance_criteria exige verificar que raiz ainda contém "RewriteBase /" |
</threat_model>
