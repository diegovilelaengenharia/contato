# PLANO 06-01: Organização do Formulário de Cliente e Validação de Fluxos

Este plano visa organizar a experiência do usuário no Painel Admin, corrigindo a ordem visual dos campos no cadastro/edição de clientes e garantindo que os fluxos de atualização de etapa estejam robustos.

## Objetivos
1. Reordenar as seções no `includes/form_cliente_template.php` para seguir uma sequência lógica (1. Acesso, 2. Dados Pessoais, 3. Endereço Residencial, 4. Dados do Imóvel, 5. Dados do Processo).
2. Garantir que todas as máscaras de input estejam consistentes.
3. Verificar a funcionalidade de "Ver Timeline" (iframe) para garantir que Diego veja exatamente o que o cliente vê.

## Passos

### 1. Reorganização do Template de Formulário
- [ ] Editar `area-cliente/includes/form_cliente_template.php`.
- [ ] Mover a seção "1. Acesso & Fotos" para o topo do formulário.
- [ ] Mover a seção "2. Dados Pessoais" logo após o acesso.
- [ ] Mover a seção "3. Endereço Residencial" após os dados pessoais.
- [ ] Mover a seção "4. Dados do Imóvel / Obra" após o endereço residencial.
- [ ] Mover a seção "5. Dados do Processo" para o final (antes dos campos extras).

### 2. Melhoria da UX de Timeline
- [ ] Verificar o arquivo `area_cliente.php` para garantir que o parâmetro `simular_timeline=1` oculte cabeçalhos/rodapés desnecessários para a visualização dentro do modal do admin.

## Verificação
1. Abrir o cadastro de um novo cliente e verificar a ordem das seções.
2. Editar um cliente existente e verificar se os dados são carregados corretamente na nova ordem.
3. Abrir o modal "Ver Timeline" no admin e verificar se a visualização está limpa.
