# PLAN 09-01: Redesign da Área Admin + Hardening

**Milestone:** v2.0 — Hardening, Auditoria e Features
**Goal:** Refazer a interface da área admin (visual feio/datado, confuso, com bugs e lento)
mantendo 100% da lógica PHP, e remover dívidas de higiene de produção.

## Contexto / Decisões do usuário (2026-05-31)

- Landing (contato) e área do cliente estão **boas** — só ajustes pontuais, NÃO refazer.
- A dor é a **área admin**: visual, organização, bugs e lentidão ("tudo").
- Redesign **do zero, em código** (HTML/CSS/PHP). Stitch MCP não está conectado neste
  ambiente, então não se usa o fluxo Stitch — design system derivado da marca
  (verde `#197e63`, fonte Outfit), revisão tela a tela, deploy para revisar.

## Escopo

1. **Fundação:** remover debug de `admin.php` (display_errors + caixa "FATAL ERROR");
   reescrever `admin_style.css` como design system coerente cobrindo TODAS as classes em uso.
2. **Shell:** `sidebar.php` + layout do `admin.php` sem estilos inline; toggle mobile.
3. **Telas:** `dashboard.php` (Visão Geral), `timeline.php`, `pendencias.php`,
   `financeiro.php` (+ helper `renderFinTable`), `documentos.php`, `arquivos.php`.
4. **Compat:** estilos limpos de top-nav/`.btn-menu` para `gerenciar_cliente.php`,
   `admin_config.php` e `avisos_gerais.php` (que usam `includes/ui/header.php`).
5. **Hardening de quebra:** corrigir N+1 no dashboard, `require sidebar_widgets` duplicado.

## Requirements tocados

- ADM-14 (interface admin funciona bem em desktop e mobile) — endereçado pelo design system responsivo.
- SEC-04 (remover debug de produção) — parcial: `admin.php` limpo; scripts soltos tratados na limpeza seguinte.

## Restrições

- Sem mudança de stack, sem frameworks/bundlers JS.
- Não tocar em Auth/CSRF/PDO nem nos endpoints de `actions/`.
- Manter os nomes de classes que as views já usam.
