## Design System

| Property | Value |
|----------|-------|

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon internal gaps, badge inline padding |
| sm | 8px | Tight spacing: heading gap, badge-to-text gap, button icon gap |
| md | 16px | Default element spacing: card inner gap, button horizontal padding |
| lg | 24px | Section card padding (mobile), grid gap between cards |
| xl | 32px | Section card padding (desktop), major block gaps |
| 2xl | 48px | Section-to-section vertical gap (`content` grid gap) |
| 3xl | 64px | Top-of-page hero padding (desktop only) |

Exceptions:
- Touch targets (floating buttons, hero CTA buttons): minimum 48px height
- Step indicator circles (como-funciona): 48px diameter
- Diego photo (sobre section): 120px circle mobile, 160px desktop

**Source:** Existing `style.css` patterns (hero padding 36px/40px, content gap 48px, link-card padding 16px/20px)

### "Como Funciona" Section (new)

**Position:** After `</header>`, before `<main class="content">`. New `<section id="como-funciona">`.

**Structure:**
```
<section id="como-funciona" class="como-funciona-section reveal">
  <h2 class="section-heading">Como Funciona</h2>
  <div class="steps-grid">
    [step 1] [step 2] [step 3]
  </div>
</section>
```

**Step component structure:**
```html
<div class="step">
  <div class="step-indicator">
    <span class="step-number">1</span>
  </div>
  <div class="step-body">
    <span class="material-symbols-rounded step-icon" aria-hidden="true">{icon}</span>
    <p class="step-label">{label}</p>
    <p class="step-desc">{description}</p>
  </div>
</div>
```

**Step indicator:** 48px circle, `background: var(--color-primary)`, white text, font-size 1.25rem weight 700, centered.
Between steps on desktop: connector line `2px solid rgba(19,111,92,0.2)`, `flex: 1`, vertically centered at step-indicator mid-height.
On mobile: no connector line (steps stack vertically, connector replaced by implicit vertical spacing).

**Layout:**
- Mobile (default): `display: flex; flex-direction: column; gap: 24px`
- Desktop (min-width 640px): `display: flex; flex-direction: row; align-items: flex-start; gap: 0` with connector lines between steps

**Section card:**
- `background: var(--color-surface)`
- `border-radius: var(--radius-large)` (16px)
- `box-shadow: var(--shadow-soft)`
- Mobile padding: 24px
- Desktop padding: 32px 40px

**Step icon names (Material Symbols Rounded):**
- Step 1: `forum` (chat/conversation)
- Step 2: `search` (analysis)
- Step 3: `verified` (completion/care)

**Step content:**

| Step | Label | Description |
|------|-------|-------------|
| 1 | Me chama no WhatsApp | Me conta o que precisa — regularização, aprovação ou projeto. Sem compromisso. |
| 2 | Analiso seu caso | Avalio a situação do imóvel e indico o caminho mais rápido e econômico. |
| 3 | Cuido de tudo | Executo cada etapa — da prefeitura ao cartório — até seu imóvel estar 100% regularizado. |

**Scroll reveal:** `.reveal` class on `<section>`. Uses existing IntersectionObserver system.

**Source:** D-10 (section exists, 3 steps), D-11 (planner defines visual and texts — this spec defines both)

## Interaction Contracts

### Scroll Reveal

Apply `.reveal` class (existing IntersectionObserver system — `.reveal` + `.reveal.active`) to:
- `<section id="como-funciona">` — section-level reveal
- Each `<section class="links">` — if not already present
- `<section id="sobre">` — section-level reveal

Do NOT apply `.reveal` to individual `.step` elements — the parent section reveal is sufficient for visual clarity.

The existing JS IntersectionObserver code handles this automatically via `document.querySelectorAll('.reveal')`.

### Modal Preservation (orçamento)

The `#serviceModal` and all associated JS (`openModal()`, `closeModal()`, `servicesData`, `whatsappSubmitBtn`) must be kept **exactly as-is**. No modifications.

The `.link-card` `onclick="openModal('Category')"` handlers remain unchanged.

### bioModal Removal

`#bioModal` div and `openBioModal()` / `closeBioModal()` JS functions are removed.
Footer button `onclick="openBioModal()"` is replaced with anchor `href="#sobre"`.
Hero `<a href="#sobre" class="name-link">` anchor already exists — no change needed.

### Floating Buttons

Keep existing `.floating-buttons` div and both floating buttons (WhatsApp and Instagram) exactly as positioned:
- `position: fixed; bottom: 24px; right: 24px`
- WhatsApp: `pulse-soft` animation (existing)
- Instagram: `float-vertical` animation with 0.5s delay (existing)

No repositioning. The planner discretion (D-20) is resolved: keep existing positioning — it works well with the one-page layout and does not overlap critical content.

### Navigation Anchors

The one-page scroll uses anchor links. No JavaScript smooth-scroll needed — use CSS:
```css
html {
  scroll-behavior: smooth;
}
```
Add this to `style.css` if not already present.

Anchor targets:
- `#hero` — top of page (logo/header)
- `#como-funciona` — como funciona section
- `#sobre` — sobre section (also targeted by footer "Sobre Mim" link)
